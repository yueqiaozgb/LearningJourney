# MYSQL

MySQL是一个开源的数据库管理系统，它最大的优势就是免费，无论是学习还是工作，打交道最多的还是MySQL，作为程序员安生立命的根本技能说也不为过，在面试中，MySQL也是考察的重点，在大学的课程中，学习到的知识点包括SQL语句、范式以及事务，但是对于其底层的具体实现，了解甚少

**SQL函数：**

对于SQL语句的学习，主要集中在函数的运用，事实上，一些基本的增删改查已经熟练掌握，但是一些新的函数，比如日期函数、计算函数以及分组函数使用的场景不多，并不熟练

**数据引擎以及索引：**

其次是对于数据库的引擎类型，InnoDB和MyISAM，前者支持事务，是在业务系统中使用最多的引擎，后者不支持事务，相应的，读写速度也更快，对于一些无需修改的配置，可以考虑存储到MyISAM中，而业务数据，多数使用前者

其次是索引类型，有七类，常见的有**主键索引**、**普通索引**、**唯一索引**及**组合索引**，除此之外还有**全文索引**、**前缀索引**和**空间索引**

**锁以及数据存储：**

对于锁的考察也是重中之重，很多时候，读写锁带来的bug不仅严重，还难以排查

从颗粒度来说，有**表锁**和**行锁**，从锁级别来说，有**共享锁**和**排他锁**，也就是**读锁和写锁**，从按照使用方式来说，有**乐观锁**和**悲观锁**

数据存储主要涉及到索引和数据的存储，索引使用的是B+树，在 MySQL 中，B+ 树索引按照存储方式的不同分为**聚集索引**和**非聚集索引**

- 聚集索引（聚簇索引）：

  以 InnoDB 作为存储引擎的表，表中的数据都会有一个主键，即使你不创建主键，系统也会帮你创建一个隐式的主键

  这是因为 InnoDB 是把数据存放在 B+ 树中的，而 B+ 树的键值就是主键，在 B+ 树的叶子节点中，存储了表中所有的数据

  这种以主键作为 B+ 树索引的键值而构建的 B+ 树索引，我们称之为聚集索引

- 非聚集索引（非聚簇索引）：

  以主键以外的列值作为键值构建的 B+ 树索引，我们称之为非聚集索引

  非聚集索引与聚集索引的区别在于非聚集索引的叶子节点不存储表中的数据，而是存储该列对应的主键，想要查找数据我们还需要根据主键再去聚集索引中进行查找，这个再根据聚集索引查找数据的过程，我们称为回表

明白了聚集索引和非聚集索引的定义，我们应该明白这样一句话：**数据即索引，索引即数据**
