# 关于语言的选择

- ### **Go (Golang)**

  #### 优势：

  1. **并发性强**：Go 的 Goroutines 和 Channel 原生支持并发，适合开发高并发的网络服务（如微服务架构）。
  2. **跨平台编译**：Go 支持轻松跨平台编译，特别适合需要部署到多平台的项目。

  #### 适用场景：

  - 高并发系统（如微服务、API 网关）
  - 云原生开发（如 Kubernetes 扩展）
  - 工具开发（如 CLI 工具或 DevOps 工具）

  ------

  ### **Rust**

  #### 优势：

  1. **性能极高**：接近 C/C++ 的效率，但提供更好的内存安全。
  2. **内存安全**：通过所有权系统，避免了常见的内存管理问题（如空指针和数据竞争）。
  3. **生态崛起**：在系统编程和 WebAssembly 领域逐渐占据一席之地，尤其适合开发高性能系统和嵌入式设备。

  #### 适用场景：

  - 高性能服务（如游戏引擎、数据库）
  - 系统编程（如操作系统、嵌入式开发）
  - WebAssembly（如前端与后端的跨语言交互）

Golang更偏向于业务，Rust更偏向于底层开发，从未来的角度来说，Golang和Java较为重合，可以考虑Rust
